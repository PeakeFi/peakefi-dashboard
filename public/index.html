API Integration Notes for Cameron

Wire this dashboard to Double's API using the endpoints below.

1. Authentication

Double uses OAuth2 with a client ID + client secret. Generate these in Double under Settings → API Keys (super admin required). Store these securely — the secret is only shown once. All requests need an Authorization: Bearer {token} header.

2. Pull Client List

Use the Swagger docs at api.doublehq.com/api-docs to find the clients endpoint. You'll need: client name, preparer, reviewer. This should mirror what you already pull for the Client Database tool.

3. Pull "Classify Transactions" Tasks

Filter tasks by name = "Classify Transactions". You need: due date, status, last completed date. The key calculation is daysRemaining = dueDate - today. Status logic: ≥8 days = On Track (green), 4–7 days = Due Soon (yellow), 1–3 days = Urgent (orange), ≤0 days = Overdue (red).

4. Sync Mechanism

The "Sync from Double" button should call the API, refresh the data, and update the table. Consider caching with a 15-minute TTL so you're not hammering the API. Display last sync time in the header. If you want auto-refresh, a 30-minute interval is reasonable.

5. Data Shape (Replace Mock Data)

Swap the MOCK_DATA array in the script with your API response. Each entry needs: 

{
  client: "Client Name",
  preparer: "Preparer Name",
  reviewer: "Reviewer Name",
  lastClassified: "2026-03-01",  // date task was last completed
  dueDate: "2026-03-15"          // next due date (lastClassified + 14 days)
}
